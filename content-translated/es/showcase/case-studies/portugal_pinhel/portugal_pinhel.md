# QGIS y el Mapeo de Riesgo de Incendio Forestal en Portugal
Portugal se ha visto afectado por grandes incendios forestales que causaron grandes pérdidas, no solo ambientales, sino también económicas y sociales. Para enfrentar este riesgo, el gobierno portugués, a través de la Autoridad Nacional Forestal (AFN), decidió, hace unos años, crear oficinas técnicas asignadas a los gobiernos locales que, entre otras habilidades, deben preparar Planes Municipales para la Protección contra Incendios Forestales y la Respuesta Operativa.

In order to support the elaboration of these documents, the AFN provided technical guides with a methodology for calculating and mapping the Forest Fire Hazard and Risk. Moreover, tutorials have been offered to follow this methodology, using Proprietary Software. However, the acquisition and licensing costs of that software are unaffordable for most of the smaller local administrations and so, it was decided to create and make available a guide with a methodology for developing Hazard and Risk cartography using only Free and Open Source Software \[<http://goo.gl/TSv2E>\].

Se propuso utilizar el siguiente paquete de software: bibliotecas QGIS, GRASS GIS, gvSIG y GDAL / OGR. Esa guía describe todas las tareas de geoprocesamiento necesarias para la elaboración de los planes de Protección contra incendios forestales y Respuesta operativa, de acuerdo con la metodología AFN. Una gran parte de las tareas se desarrollaron con QGIS, y el análisis espacial en el modelo ráster se desarrolló con GRASS. gvSIG se utilizó para el análisis de red, con su extensión de Análisis de red, y las bibliotecas GDAL / OGR se utilizaron para las transformaciones entre los sistemas de referencia de coordenadas.

Después de un extenso juego de pruebas a la metodología, y tres años de aplicación real en la preparación del Plan Operativo para el municipio de Pinhel, se puede decir que las alternativas propuestas permiten reemplazar, con muchas ventajas, el Software Privativo que generalmente solía llevar a cabo esta tarea. La validación de los resultados muestra que, a pesar de la relativa simplicidad del modelo conceptual, su capacidad predictiva es bastante buena, y que la implementación del modelo en software de código abierto no interfiere negativamente con los resultados, sino todo lo contrario.

En una segunda fase, se intentó acelerar este proceso, utilizando modelos para establecer un flujo de trabajo que realizara un amplio conjunto de tareas, casi sin intervención humana. Esta segunda etapa consistió esencialmente en la automatización de todo el procedimiento descrito en la guía práctica resultante de la primera fase. Dicha automatización podría significar una reducción de varias horas de intenso trabajo por parte del técnico asignado a producir anualmente Planes de defensa forestal, durante solo unos minutos, en los que la intervención humana se reduce a la selección de datos de entrada y la indicación del lugar donde pretendemos guardar los datos de salida.

![Interface of the Sextante Model to calculate the Probability of the Forest Fire Hazard.](./images/portugal_pinhel3.png){.align-right}

In furtherance of this second phase, we used the Python version of Sextante software, that works integrated into QGIS and adds a broad set of independent applications (GRASS GIS, SAGA GIS, OTB, R, GDAL/OGR, Pymorph, LASTools, Python scrips, etc..) in a single interface, providing a huge geoprocessing toolbox to QGIS users. Besides the integration of these applications in QGIS, Sextante has a tool for creating models, taking advantage of the modules offered by any of those softwares which aggregates. So, we\'ve created a model to automate the process of producing Forest Fire Hazard and Risk maps, using GRASS, SAGA, fTools and MMQGIS tools.

![Part of the model developed for automation of the production of cartography for Forest Fire Hazard and Risk.](./images/portugal_pinhel4.png){.align-right}

The results obtained so far are very promising, as already can be automatically achieved the creation of the Hazard and Risk Maps \[2\]. Taking into account that the Python version of Sextante is still very recent and is in heavy construction, there are some problems that must be corrected so that the models may be completed, which certainly will happen very soon, given the momentum that Sextante project presents. As future work, we intend to apply, also automatically, the symbology to the results as well as provide the final layouts in QGIS Composer, ready for export and/or print. Upon completion of the second phase and the realization of a sufficient set of tests that can validate the results obtained, it is our intention to provide the model free and openly.
## Conclusiones
Las alternativas estudiadas demostraron ser de enorme calidad, permitiendo todas las operaciones recomendadas por las guías técnicas de la Autoridad Forestal Nacional, de muchas maneras, de manera más eficiente que con el software patentado. En términos de facilidad de uso, se observó que este tipo de software no es, de ninguna manera, más complejo que el software privativo, ya que solo requiere un mayor conocimiento técnico de los modelos y algoritmos implementados que, sin embargo, permiten un mayor grado de libertad , haciendo posible explorar y ajustar los modelos a cada situación particular. El proceso de producción de cartografía de riesgo y riesgo de incendios forestales utilizando, exclusivamente, software de código abierto está totalmente consolidado, después de varios años de pruebas y aplicación en el municipio de Pinhel.

El hecho de que el software de código abierto se base en estándares y soporta la mayoría de los formatos de datos abiertos, permite la interoperabilidad completa entre el software, permitiendo al usuario optar por el más adecuado en cada momento. A pesar de que nuestra propuesta apunta a un conjunto específico de software, nada impide ser reemplazado por alguna de las alternativas existentes en la amplia gama de propuestas de Software Libre y de Código Abierto para Geoespacial. Sin embargo, QGIS se presenta cada vez más como la solución FOSS4G más completa, estable y fácil de usar, y cuyo proyecto es más dinámico, con una rápida corrección de errores y con la implementación casi diaria de nuevos complementos que agrega funcionalidades específicas a las más diversas áreas de actividades.
