# Використання QGIS у просторовій інфраструктурі м. Устер
м. Устер використовує QGIS з 2007 року. Рішення кантону Золотурн використовувати QGIS в якості заміни ArcView надихнуло м. Устер спробувати її. Місто приєдналось до зусиль з підтримки процесу розробки QGIS шляхом внесення пропозиції, тестування нового функціоналу та надання фінансової підтримки. В свою чергу місто отримало багатоплатформову настільну ГІС та веб-ГІС, яка швидко розвивається та супроводжується світовим співтовариством користувачів та розробників.

![](./images/suisse_uster1.png){.align-right}

В якості сховища даних використовується PostgreSQL/PostGIS. Інші бази та джерела даних регулярно реплікуються у PostgreSQL за допомогою таких інструментів як OGR та FME. Спочатку QGIS в основному використовувалась лише для перегляду даних. Першим користувачем QGIS став фермер, який повинен був переглядати заявки інших фермерів на отримання сільськогосподарських субсидій. Згодом, в процесі вдосконалення інструментів редагування та редактора макетів, QGIS почала використовуватись для редагування даних та публікації карт. Разом з іншими швейцарськими користувачами QGIS (Золотурн, Тун та CDE (Університет Берна)) ми координували та фінансово підтримували вдосконалення цих інструментів разом з іншими вдосконаленнями в оздобленні, яке залежить від даних, автоматичному підписуванні, управлінні атрибутами та формами.

![](./images/suisse_uster2.png){.align-right}

Адміністрація міста також використовує власницькі ГІС: Autodesk Topobase для картографування та редагування кадастрових даних та даних про інфраструктуру стічних вод, GE Smallworld для управління газовими, водними та електричними мережами. Однак ці експертні системи використовуються лише фахівцями. Всі інші ГІС-проекти та перегляд даних виконуються в QGIS.

In the second half of 2010, first tests with QGIS server were carried out. In 2011 we introduced a new WebGIS system based on the QGIS WMS server and GeoExt/OpenLayers as the web client. The main reasons for the decision to use QGIS server were the considerable time savings made possible by re-using the existing QGIS desktop project configurations, rather than having to manage parallel Web project configurations. Additional advantages are the rich set of symbolization and labeling options and the consistent map rendering between the Desktop GIS application and the Web-GIS. The QGIS webclient can be tested and used at <http://gis.uster.ch/>

![](./images/suisse_uster3.png){.align-right}

Сьогодні QGIS використовується у наступних ГІС-проектах:
- перегляд даних по стічним водам, інтеграція додаткових баз даних та прямий доступ до відео камер в каналах та протоколів перевірки
- перегляд та редагування даних моніторингу охорони навколишнього середовища та міського планування стічних вод та гідросистем (німецькою: Genereller Entwässerungsplan)
- перегляд даних підземного кадастру (стічні води, мережі водо-, газо- та електропостачання, згодом і комунікаційні мережі) (німецькою: Leitungskataster)
- перегляд даних кадастру та польових робіт
- перегляд та редагування реєстру природоохоронних територій
- перегляд та редагування даних міського та регіонального планування і розвитку
- перегляд та редагування даних дорожньої інфраструктури
- перегляд статистики, що генерується з бази даних поточних та колишніх мешканців
- планування та візуалізація енергетичних ресурсів (газоносні райони, тепло від стічних вод, приповерхнева геотермальна енергія, блоки теплових станцій тощо)

Багато інших проектів знаходиться в розробці.

![](./images/suisse_uster4.png){.align-right}

У 2007 році було не зовсім зрозуміло якими темпами буде вестись розробка QGIS та як розвиватимуться суспільна та комерційна підтримки. Сьогодні можна впевнено говорити, що швидкість розробки та вдосконалення, зроблені співтовариством QGIS перевершили наші очікування. Некомерційна та комерційна підтримка від компаній Sourcepole та Norbit знаходяться на відмінному рівні. Існує прямий зв'язок з розробниками без перешкод з боку відділів маркетингу та продажу. Розробники цих двох компаній також є провідними розробниками проекту QGIS та мають чудове знання кодової бази.

Після повідомлення помилки часто виправляються протягом годин або днів, а нові можливості реалізуються у стислий термін. Новий функціонал можна спробувати одразу після його реалізації та запровадити як тільки вони стануть достатньо стабільними для повсякденного використання. Відпадає необхідність в очікувані офіційного випуску. Ми насолоджуємось свободою та можливістю впливати на розвиток проекту, можливість встановлювати програму на будь-яку кількість комп'ютерів без необхідності ліцензування. Ми з нетерпінням чекаємо на подальшу плідну співпрацю зі спільнотою QGIS та нашими комерційними партнерами в просуванні проекту QGIS.
