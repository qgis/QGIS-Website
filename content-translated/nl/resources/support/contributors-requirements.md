# Vereisten bijdragen
## Vereisten voor nieuwe deelnemers
We verwelkomen nieuwe deelnemers aan onze gemeenschap - we willen dat u QGIS gebruikt als een platform om uw werk te doen - en we geloven dat QGIS daar een fantastisch platform voor is. Alle ontwikkelaars/teamleden moeten, om code, patches, of documentatie te kunnen bijdragen aan het project QGIS:
- Zich houden aan het [Social Contract](#social-contract).
- Bekend zijn met het proces voor ontwikkeling en het verwijzingsmateriaal dat beschikbaar is in de [Developers Guide for QGIS](https://docs.qgis.org/testing/en/docs/developers_guide/index.html), en het [document CODING](https://github.com/qgis/QGIS/blob/master/CONTRIBUTING.md) in de bron van de Sources van QGIS.
- Zorg ervoor dat alle aan de opslagplaats toegevoegde code onbelast is en compatibel met de [licentie van QGIS]({{< relref "license" >}}), GNU Public License (GPL) Versie 2 of enige latere versie.
- Geen code gebruiken of opnemen die een inbreuk veroorzaakt op handelsmerken of een patent. Als u er niet zeker van bent, vraag het dan na op de [developer mailinglijst]({{< relref "community/organisation/mailinglists/#qgis-developers-list" >}}) of neem contact op met een lid van het [Project Steering Committee]({{< relref "community/organisation/#project-steering-committee" >}}).
- Neem deel aan de [qgis-developer mailinglijst]({{< relref "community/organisation/mailinglists/#qgis-developers-list" >}}) en draag bij aan de besprekingen
- Deze vereisten zijn zowel van toepassing op ontwikkelaars die commits willen indienen als op hen die patches indienen.
- Lezen en instemmen met de [Gedragscode]({{< relref "conduct" >}}).
- Lees en stem in met de [Standaarden voor coderen](https://docs.qgis.org/testing/en/docs/developers_guide/codingstandards.html).
- Lezen en instemmen met de [Verklaring over diversiteit]({{< relref "diversity" >}}).
- Instemmen met deze vereisten door een e-mail te sturen aan de [qgis-developer mailinglijst]({{< relref "community/organisation/mailinglists/#qgis-developers-list" >}}) waarin de instemming wordt verwoord.

## Sociaal contract
QGIS is een samenwerkingsproject. Als een open-sourceproject hopen we op en zoeken deelnemen aan en bijdragen van code van geïnteresseerde partijen. Echter, om hoge standaarden te behouden, raden we aan om deel te nemen in het project overeenkomstig het volgende sociale contract:
- Wees beleefd tegen alle leden van het project
- Zorg ervoor dat alle code die u indient via een patch of commit juist is gedocumenteerd om de bron te kunnen controleren
- Zorg ervoor dat alle code die u indient via een patch of commit voldoet aan onze CodingStandards
- Geen code in te dienen zondere toestemming om die code in te mogen dienen
- Geen code in te dienen die de GNU Public License schendt
- De hierboven vermelde [Vereisten deelnemers](#contributor-requirements) te volgen
- Zorg ervoor (naar uw beste kunnen) dat commits die u indient bij git de repository in een compileerbare staat laten
- Zorg ervoor dat een nieuw aan de bron toegevoegde klasse wordt vergezeld door een testeenheid
- Toepasselijke testen uit te voeren voordat nieuwe code wordt ingediend bij de repository
- Te glimlachen, te lachen en af en toe een mop te vertellen - we zijn hier allemaal om lol te hebben!
